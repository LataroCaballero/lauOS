<task type="checkpoint:human-action">
  <name>Task 1: DNS, VPS binary, and systemd setup</name>
  <files>/home/pocketbase/pb/pocketbase, /lib/systemd/system/pocketbase.service</files>
  <action>
    Execute the following steps on the VPS via SSH. All commands assume Debian/Ubuntu-based OS.

    **Step 1 — Create subdomain DNS A record**
    In your domain registrar DNS panel, create an A record:
    - Name: pb (or your chosen subdomain prefix)
    - Value: VPS public IP address
    - TTL: 300
    Verify propagation before proceeding: `dig pb.yourdomain.com +short` must return your VPS IP.

    **Step 2 — Create a dedicated service user**
    ```bash
    sudo useradd --system --no-create-home --shell /bin/false pocketbase
    sudo mkdir -p /home/pocketbase/pb
    sudo chown -R pocketbase:pocketbase /home/pocketbase/pb
    ```

    **Step 3 — Download PocketBase binary**
    ```bash
    cd /home/pocketbase/pb
    # Download PocketBase v0.36.6 for Linux AMD64 (adjust arch if needed)
    sudo wget https://github.com/pocketbase/pocketbase/releases/download/v0.36.6/pocketbase_0.36.6_linux_amd64.zip
    sudo apt-get install -y unzip
    sudo unzip pocketbase_0.36.6_linux_amd64.zip
    sudo rm pocketbase_0.36.6_linux_amd64.zip
    sudo chown pocketbase:pocketbase /home/pocketbase/pb/pocketbase
    sudo chmod +x /home/pocketbase/pb/pocketbase
    ```

    **Step 4 — Create systemd unit file**
    Create `/lib/systemd/system/pocketbase.service` with this exact content:
    ```ini
    [Unit]
    Description=PocketBase
    After=network.target

    [Service]
    Type=simple
    User=pocketbase
    Group=pocketbase
    LimitNOFILE=4096
    Restart=always
    RestartSec=5s
    StandardOutput=append:/home/pocketbase/pb/pb.log
    StandardError=append:/home/pocketbase/pb/pb.log
    WorkingDirectory=/home/pocketbase/pb
    ExecStart=/home/pocketbase/pb/pocketbase serve --http="127.0.0.1:8090"

    [Install]
    WantedBy=multi-user.target
    ```

    **Step 5 — Enable and start the service**
    ```bash
    sudo systemctl daemon-reload
    sudo systemctl enable pocketbase
    sudo systemctl start pocketbase
    sudo systemctl status pocketbase
    ```
    Expected: status shows `active (running)`.

    **Step 6 — Verify localhost binding only**
    ```bash
    ss -tlnp | grep 8090
    ```
    Expected: output shows `127.0.0.1:8090` — NOT `0.0.0.0:8090`.
  </action>
  <how-to-verify>
    1. `sudo systemctl status pocketbase` shows `active (running)`
    2. `ss -tlnp | grep 8090` shows `127.0.0.1:8090`
    3. `curl http://127.0.0.1:8090/api/health` returns `{"code":200,"message":"API is healthy.",...}`
    4. `dig pb.yourdomain.com +short` returns your VPS IP
  </how-to-verify>
  <resume-signal>Type "infra-ready" once PocketBase is running and DNS is propagated, or describe any errors</resume-signal>

<task type="checkpoint:human-action">
  <name>Task 2: Nginx TLS and PocketBase admin setup</name>
  <files>/etc/nginx/sites-available/pb.yourdomain.com</files>
  <action>
    **Step 1 — Install Nginx and Certbot**
    ```bash
    sudo apt-get update
    sudo apt-get install -y nginx certbot python3-certbot-nginx
    ```

    **Step 2 — Create Nginx site config**
    Create `/etc/nginx/sites-available/pb.yourdomain.com` with this content (replace `pb.yourdomain.com` with your actual subdomain):
    ```nginx
    server {
        listen 80;
        server_name pb.yourdomain.com;
    }
    ```

    **Step 3 — Enable the site**
    ```bash
    sudo ln -s /etc/nginx/sites-available/pb.yourdomain.com /etc/nginx/sites-enabled/
    sudo nginx -t
    sudo systemctl reload nginx
    ```

    **Step 4 — Obtain TLS certificate**
    Wait until `dig pb.yourdomain.com +short` returns your VPS IP, then:
    ```bash
    sudo certbot --nginx -d pb.yourdomain.com
    ```
    When prompted, choose option to redirect HTTP → HTTPS. Certbot will automatically update the Nginx config with SSL directives and set up auto-renewal.

    **Step 5 — Update Nginx config for PocketBase proxy**
    After certbot runs, edit `/etc/nginx/sites-available/pb.yourdomain.com`. The SSL server block should contain:
    ```nginx
    server {
        listen 443 ssl;
        server_name pb.yourdomain.com;
        client_max_body_size 10M;

        ssl_certificate /etc/letsencrypt/live/pb.yourdomain.com/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/pb.yourdomain.com/privkey.pem;
        include /etc/letsencrypt/options-ssl-nginx.conf;
        ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

        location / {
            proxy_set_header Connection '';
            proxy_http_version 1.1;
            proxy_read_timeout 360s;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_pass http://127.0.0.1:8090;
        }
    }
    ```

    ```bash
    sudo nginx -t && sudo systemctl reload nginx
    ```

    **Step 6 — PocketBase Admin UI first-time setup**
    Visit `https://pb.yourdomain.com/_/` in your browser. Create the superadmin account (email + password). Store credentials securely.

    **Step 7 — Configure PocketBase settings**
    In PocketBase Admin UI > Settings:
    - **Application name:** lauOS
    - **Application URL:** https://pb.yourdomain.com
    - **Token options > Auth token duration:** Set to 31536000 (365 days in seconds) to ensure "never expires unless logout" UX even without daily visits
    - **Allowed origins (CORS):** Add `http://localhost:3000` (dev) — production Vercel URL will be added after deploy in Phase 3+

    **Step 8 — Reboot test**
    ```bash
    sudo reboot
    ```
    Wait ~2 minutes, then SSH back and verify:
    ```bash
    sudo systemctl status pocketbase
    ```
    Expected: `active (running)`.

    **Step 9 — Local Mac dev setup**
    On your Mac:
    ```bash
    mkdir -p ~/pb
    # Download macOS binary from https://github.com/pocketbase/pocketbase/releases/tag/v0.36.6
    # Extract pocketbase binary to ~/pb/
    chmod +x ~/pb/pocketbase
    # Run local PocketBase for development:
    ~/pb/pocketbase serve
    # Access local Admin UI at http://127.0.0.1:8090/_/
    ```
    Create a local superadmin account at `http://127.0.0.1:8090/_/`. This is your development PocketBase instance.
  </action>
  <how-to-verify>
    1. `https://pb.yourdomain.com/_/` loads the PocketBase Admin UI in browser with a valid TLS cert (padlock icon)
    2. After reboot: `sudo systemctl status pocketbase` shows `active (running)`
    3. `certbot renew --dry-run` completes without errors (Certbot auto-renewal works)
    4. Local Mac: `http://127.0.0.1:8090/_/` loads the Admin UI
  </how-to-verify>
  <resume-signal>Type "infra-complete" once TLS is working and reboot test passes, or describe any errors</resume-signal>
