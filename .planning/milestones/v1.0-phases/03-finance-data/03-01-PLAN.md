**A. Create `lauos/src/lib/money.ts`:**

```typescript
/**
 * Monetary conversion helpers for the Finance module.
 * All amounts stored as integer centavos (×100). Exchange rates stored ×10000.
 * Never store floats. Always use Math.round() when converting from display.
 */

/** Convert user-entered display string to integer centavos. "1234.56" → 123456 */
export const toCentavos = (display: string): number =>
  Math.round(parseFloat(display) * 100)

/** Convert integer centavos to locale-formatted display string. */
export const fromCentavos = (centavos: number, currency: 'ARS' | 'USD'): string => {
  const value = centavos / 100
  if (currency === 'ARS') {
    return value.toLocaleString('es-AR', { style: 'currency', currency: 'ARS' })
  }
  return value.toLocaleString('en-US', { style: 'currency', currency: 'USD' })
}

/** Convert display exchange rate string to stored integer (rate × 10000). "1420.50" → 14205000 */
export const toRateStored = (rate: string): number =>
  Math.round(parseFloat(rate) * 10000)

/** Convert stored rate integer back to display string with 2 decimal places. */
export const fromRateStored = (stored: number): string =>
  (stored / 10000).toFixed(2)
```

**D. Create `pocketbase/pb_migrations/1773191964_create_transactions.js`:**

The transactions collection depends on accounts and categories existing. Field notes:
- `amount_centavos`: onlyInt: true, min: 1 — rejects decimals and zero
- `exchange_rate_stored`: onlyInt: true, not required — only populated for USD-account transactions. Stores rate × 10000 (e.g., 1420.50 → 14205000)
- `transfer_pair_id`: text, max: 30, optional — holds the ID of the paired transfer record; set on both legs
- `category`: relation to categories, NOT required at DB level (transfers don't need one)
- `date`: date field for the user-specified transaction date (not the auto-created timestamp)

Create `pocketbase/pb_hooks/seed_categories.pb.js`. This hook fires after a new user record is successfully persisted and creates 10 default category records owned by that user.

**CRITICAL implementation notes:**
- Use `onRecordAfterCreateSuccess`, NOT `onRecordBeforeCreateRequest` (deprecated in PocketBase 0.23+)
- The callback MUST call `e.next()` at the end — failing to do so breaks the event chain
- `new Record(collection)` and `e.app.save(record)` are the correct patterns per PocketBase JS Records docs
- The hook is scoped to the `"users"` collection (second argument to the function)
- If the categories collection doesn't exist yet (migration not applied), wrap in try/catch so hook doesn't crash on fresh installs during migration order edge cases

```javascript
/// <reference path="../pb_data/types.d.ts" />

onRecordAfterCreateSuccess((e) => {
  try {
    const categoriesCollection = e.app.findCollectionByNameOrId("categories")

    const defaults = [
      { name: "Vivienda",         icon: "🏠", color: "#6366f1" },
      { name: "Alimentación",     icon: "🛒", color: "#22c55e" },
      { name: "Transporte",       icon: "🚗", color: "#f59e0b" },
      { name: "Salud",            icon: "💊", color: "#ef4444" },
      { name: "Sueldo",           icon: "💼", color: "#3b82f6" },
      { name: "Entretenimiento",  icon: "🎮", color: "#a855f7" },
      { name: "Educación",        icon: "📚", color: "#14b8a6" },
      { name: "Ropa",             icon: "👕", color: "#f97316" },
      { name: "Servicios",        icon: "⚡", color: "#64748b" },
      { name: "Otros",            icon: "📦", color: "#94a3b8" },
    ]

    for (const cat of defaults) {
      const record = new Record(categoriesCollection)
      record.set("user", e.record.id)
      record.set("name", cat.name)
      record.set("icon", cat.icon)
      record.set("color", cat.color)
      e.app.save(record)
    }
  } catch (err) {
    // Log but don't block user creation if categories collection is missing
    console.error("seed_categories hook error:", err)
  }

  e.next()
}, "users")
```
